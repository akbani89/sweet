# GlucoSnap API

FastAPI backend for the GlucoSnap glucose monitoring app.
Scan a glucometer photo ‚Üí OCR ‚Üí rules engine ‚Üí plain-English insight ‚Üí stored in PostgreSQL.

---

## Stack

| Layer        | Tech                            |
|--------------|---------------------------------|
| API          | FastAPI + Uvicorn               |
| Database     | PostgreSQL (SQLAlchemy ORM)     |
| OCR          | Tesseract + OpenCV              |
| Storage      | Local disk (default) or AWS S3 / Cloudflare R2 |
| Auth         | OTP ‚Üí JWT (RS256)               |
| Deployment   | Docker ‚Üí Render                 |

---

## Local development

### Prerequisites
- Docker + Docker Compose

### Run

```bash
git clone https://github.com/your-org/glucosnap.git
cd glucosnap

cp .env.example .env          # edit if needed (DEV_MODE=true by default)

docker compose up --build
```

API is live at **http://localhost:8000**
Interactive docs at **http://localhost:8000/docs**

---

## Deploy to Render (one-click)

1. Push this repo to GitHub.
2. Go to [render.com](https://render.com) ‚Üí **New ‚Üí Blueprint**.
3. Connect your GitHub repo.
4. Render reads `render.yaml` and provisions:
   - A **Web Service** (Docker, free tier)
   - A **PostgreSQL** database (free tier)
   - Auto-generates `SECRET_KEY`
5. Hit **Apply** ‚Äî done. First deploy takes ~5 minutes (Tesseract install).

### Environment variables (set in Render dashboard)

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Auto-set | Injected from the managed DB |
| `SECRET_KEY` | Auto-set | Auto-generated by Render |
| `DEV_MODE` | No | Set `false` in production |
| `STORAGE_BACKEND` | No | `local` (default) or `s3` |
| `AWS_ACCESS_KEY_ID` | S3 only | AWS / R2 access key |
| `AWS_SECRET_ACCESS_KEY` | S3 only | AWS / R2 secret |
| `AWS_S3_BUCKET` | S3 only | Bucket name |
| `R2_ENDPOINT_URL` | R2 only | `https://<account>.r2.cloudflarestorage.com` |

## API reference

All authenticated endpoints require:
```
Authorization: Bearer <token>
```

### Auth

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/request-otp` | Send OTP to phone number |
| `POST` | `/auth/verify-otp` | Verify OTP ‚Üí receive JWT |

### Glucose

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/glucose/scan` | Upload glucometer photo ‚Üí OCR |
| `POST` | `/glucose/manual` | Add reading manually |
| `GET`  | `/glucose/history` | Paginated reading history |
| `GET`  | `/glucose/history/{id}` | Single reading |
| `DELETE` | `/glucose/history/{id}` | Delete reading |

### User

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET`  | `/user/me` | Get profile |
| `PATCH` | `/user/me` | Update profile |
| `GET`  | `/user/doctor-targets` | Get doctor-set targets |
| `PUT`  | `/user/doctor-targets` | Set doctor targets |
| `DELETE` | `/user/me` | Deactivate account |

### Insights

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET`  | `/insights/summary?days=7` | Trend summary for N days |
| `GET`  | `/insights/latest-insight` | Latest reading insight (home screen) |

## Adding SMS (production)

In `app/routers/auth.py`, find the TODO comment and replace with:

```python
# Twilio
from twilio.rest import Client
client = Client(settings.TWILIO_ACCOUNT_SID, settings.TWILIO_AUTH_TOKEN)
client.messages.create(
    body=f"Your GlucoSnap code is {otp_code}",
    from_=settings.TWILIO_FROM_NUMBER,
    to=body.phone,
)
```

Add `twilio` to `requirements.txt` and the three Twilio env vars to Render.
